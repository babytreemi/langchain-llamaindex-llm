# Components
组件的概念
## Schema
整个代码库中使用的基本数据类型和模式

**Text**:当使用语言模型时，您可以通过文本与它们交互的主要接口。由于过度简化，许多模型都是“文本输入，文本输出”。因此，LangChain中的许多界面都以文本为中心。

**ChatMessages**:最终用户交互的主要界面是聊天界面。由于这个原因，一些模型提供者甚至开始以一种期望聊天消息的方式提供对底层API的访问。这些消息有一个内容字段(通常是文本)并与用户相关联。目前支持的用户是System、Human和AI
1. SystemChatMessage 指令信息
2. HumanChatMessage 人类交互信息
3. AIChatMessage ai系统信息

**Examples**：函数的输入以及预期的输出。它们可用于模型的训练和评估。
这些可以是一个模型的输入/输出，也可以是一个链条的输入/输出。这两种类型的例子都有不同的目的。模型的例子可以用来对模型进行微调。链条的例子可以用来评估端到端的链条，或者甚至可以训练一个模型来替代整个链条。

**Document**：一段非结构化数据。由 `page_content`（数据的内容）和 `matadata`（描述数据属性的辅助信息）组成。
